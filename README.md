# API отчетов МойСклад

Сервис генерирует Excel-отчеты о валовом обороте по данным боевого API МойСклад.

Текущий продовый сценарий:

1. Фронт получает список магазинов и месяцев.
2. Пользователь выбирает магазин и месяц.
3. API читает розничные продажи из МойСклад.
4. Сервис собирает Excel по шаблону.
5. Клиент скачивает готовый файл.

## Безопасность

Клиент МойСклад в проекте работает только на чтение.

Разрешены только `GET`-запросы. На уровне HTTP-клиента стоит предохранитель: `POST`, `PUT`, `PATCH`, `DELETE` и любые другие методы кроме `GET` запрещены кодом.

Категорически запрещено делать reset, rotate, revoke или перевыпуск боевого токена из этого проекта. Токен только читается из локального `.env` и используется для авторизации запросов на чтение.

Файл `.env` не коммитится.

## Запуск

```bat
run_api.bat
```

API поднимается на:

```text
http://127.0.0.1:8010
```

Перед боевым запуском в `.env` должен быть токен:

```env
MOYSKLAD_TOKEN=...
DATA_SOURCE=moysklad
```

Токен можно записать локально через:

```bat
set_moysklad_token.bat
```

## Магазины

Локальные магазины настраиваются в `stores.json`.

Сейчас привязана точка:

```text
planeta -> 5LB, 9 Мая, д.77, ТРЦ Планета
```

ID точки МойСклад:

```text
bcbff9c3-79c4-11f0-0a80-05d4001f0e49
```

Для проверки реальных торговых точек МойСклад:

```http
GET /moysklad/stores
```

Эта ручка делает только `GET /entity/retailstore`.

## Ручки API

Проверка сервиса:

```http
GET /health
```

Локально настроенные магазины:

```http
GET /stores
```

Месяцы для фронта:

```http
GET /months
GET /months?months_back=36
```

Пример ответа:

```json
{
  "months": [
    {
      "value": "2026-06",
      "label": "Июнь 2026",
      "year": 2026,
      "month": 6,
      "is_current": true
    }
  ]
}
```

Создать задачу генерации отчета:

```http
POST /reports/gross-turnover
Content-Type: application/json

{
  "store": "planeta",
  "month": "2026-05",
  "source": "moysklad"
}
```

Ответ:

```json
{
  "job_id": "abc123",
  "status": "processing",
  "store": "Планета",
  "month": "2026-05",
  "source": "moysklad",
  "status_url": "/reports/jobs/abc123",
  "download_url": "/reports/jobs/abc123/download"
}
```

Проверить статус:

```http
GET /reports/jobs/{job_id}
```

Скачать готовый Excel:

```http
GET /reports/jobs/{job_id}/download
```

Синхронная ручка для ручной проверки:

```http
GET /reports/gross-turnover?store=planeta&month=2026-05&source=moysklad
```

## Excel

Шаблоны лежат в:

```text
templates/
```

Готовые отчеты сохраняются в:

```text
generated/
```

Последний проверенный боевой отчет:

```text
generated/Форма_Отчета_о_валовом_обороте_2026_май_Планета_37c0cf8c712148ce850e4d36f93e8f8d.xlsx
```

Контрольные суммы по этому отчету:

```text
Оборот: 848 864.70
Чеки: 559
Товарные позиции: 559
```

Формат листа сохраняется как в исходном шаблоне.

## Что нужно доделать

Перед расширением на все магазины нужно получить или подтвердить реальные ID точек МойСклад для:

```text
Июнь
Высотная
```

После этого ID надо прописать в `stores.json` в поле `moysklad_retail_store_id`.
